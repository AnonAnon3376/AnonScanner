# 贡献指南

感谢你对 Anon Scanner 项目的关注！我们欢迎各种形式的贡献。

## 🤝 如何贡献

### 报告问题
- 使用 [Issues](https://github.com/yizhouliao/AnonScanner/issues) 报告bug
- 提供详细的错误信息和复现步骤
- 包含你的环境信息（Java版本、Burp Suite版本等）

### 功能建议
- 在 Issues 中提出新功能建议
- 详细描述功能需求和使用场景
- 讨论实现方案

### 代码贡献
1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建 Pull Request

## 📋 开发规范

### 代码风格
- 使用4个空格缩进
- 类名使用 PascalCase
- 方法名使用 camelCase
- 常量使用 UPPER_SNAKE_CASE
- 添加必要的注释

### 提交信息
使用以下格式：
```
类型: 简短描述

详细描述（可选）
```

类型包括：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

### 测试
- 确保代码能正常编译
- 在Burp Suite中测试功能
- 测试不同的AI模型

## 🛡️ 安全考虑

- 不要在代码中硬编码API密钥
- 注意处理用户输入的安全性
- 遵循安全编码最佳实践

## 📞 联系方式

- GitHub Issues: 技术问题和bug报告
- Email: 紧急安全问题

## 📄 许可证

通过贡献代码，你同意你的贡献将在 MIT 许可证下发布。
